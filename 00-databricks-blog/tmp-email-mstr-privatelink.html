문의 주신 MSTR-Databricks PrivateLink 연결에 대해 답변 드립니다.

1. MSTR → Databricks 쿼리 실행 방식

MSTR은 JDBC (Simba Spark JDBC Driver)를 통해 Databricks SQL Warehouse에 연결합니다.

항목내용
프로토콜JDBC (주), ODBC (보조)
드라이버Simba Spark JDBC Driver (Databricks 공식 제공)
접속 포트HTTPS 443
인증Personal Access Token (PAT) 또는 OAuth
SQL Warehouse 유형Classic / Pro / Serverless 모두 동일하게 지원

JDBC Connection String 예시:

jdbc:databricks://<workspace-host>:443;httpPath=/sql/1.0/warehouses/<warehouse-id>;SSL=1;AuthMech=3;UID=token;PWD=<personal-access-token>
핵심 포인트: MSTR은 Databricks workspace 도메인(예: adb-xxxx.cloud.databricks.com)에 HTTPS(443)로 접속하여 SQL을 보내고 결과를 받습니다. SQL Warehouse가 Classic이든 Serverless든, MSTR 입장에서 접속 경로(workspace URL)는 동일합니다. 따라서 PrivateLink 구성도 동일합니다.

2. PrivateLink 구성 — 방향별 정리

Serverless SQL Warehouse 환경 기준으로, 두 방향의 PrivateLink 구성이 다릅니다.

방향 A MSTR → Databricks (Frontend PrivateLink) — BI 연결의 핵심

Frontend PrivateLink란?
Databricks workspace에 접속하는 경로를 인터넷 대신 AWS 내부 네트워크(PrivateLink)로 전환하는 구성입니다. 일반적으로 사용자가 Databricks에 접속할 때는 인터넷을 경유하지만, Frontend PrivateLink를 구성하면 특정 VPC에서 Databricks workspace로의 트래픽이 AWS 백본 네트워크를 통해 프라이빗하게 전달됩니다.

동작 원리:
Databricks는 각 리전에 VPC Endpoint Service를 이미 운영하고 있습니다. 고객은 자신의 VPC에 이 서비스를 가리키는 Interface VPC Endpoint를 생성하면, 해당 VPC 내에서 Databricks workspace 도메인으로 향하는 트래픽이 인터넷이 아닌 VPC Endpoint의 Private IP → AWS PrivateLink → Databricks Control Plane 경로로 전달됩니다.

즉, MSTR VPC에 VPC Endpoint를 만들면, MSTR이 JDBC로 adb-xxxx.cloud.databricks.com:443에 접속할 때 인터넷을 거치지 않고 프라이빗하게 연결됩니다. SQL Warehouse가 Serverless여도 이 접속 경로는 동일합니다.
PrivateLink 유무에 따른 접속 경로 비교
PrivateLink 없음 MSTR → 인터넷 → Databricks workspace → SQL Warehouse
Frontend PrivateLink 적용 MSTR → VPC Endpoint (Private IP) → AWS PrivateLink → Databricks workspace → SQL Warehouse

구성 절차:

단계작업상세
1 MSTR VPC에서 Interface VPC Endpoint 생성 Service name (서울 리전):
com.amazonaws.vpce.ap-northeast-2.vpce-svc-0babb9bde64f34d7e
Security Group: Inbound TCP 443 허용
2 Databricks Account Console에 등록 Security → Networking → VPC endpoints → Register VPC endpoint
생성된 vpce-xxxxxxxx 등록
3 DNS 구성 MSTR VPC에서 workspace 도메인이 VPC Endpoint Private IP로 resolving 되도록
Route 53 Private Hosted Zone 설정
(기존 MSTR과 동일 DNS 환경이면 추가 설정 불필요)
추가 메일에서 문의 주신 내용에 대한 답변:

네, 말씀하신 대로 신규 MSTR VPC에서 VPC Endpoint 생성 → Databricks Security > Connectivity > VPC Endpoints에 등록만으로 연결이 가능합니다.

LGIT 측에서 별도 VPC Endpoint Service(제공자)를 만들 필요가 없습니다. Databricks가 이미 Endpoint Service를 제공하고 있기 때문입니다. 기존 MSTR 서비스가 workspace 도메인만으로 잘 동작하고 있다면, 이 구조가 이미 적용되어 있는 것이며 신규 MSTR도 동일한 방식으로 연결하면 됩니다.

방향 B Databricks Serverless → MSTR VPC (NCC 경유) — 일반적으로 불필요

이 방향은 Databricks Serverless에서 MSTR VPC 내부 리소스에 접근해야 할 때만 필요합니다.
일반적인 BI 연결(MSTR이 쿼리를 보내고 결과를 받는 구조)에서는 방향 A만으로 충분합니다.

참고용으로 구성 방법을 정리하면:

단계구성 주체작업
1 MSTR VPC 측 NLB(Network Load Balancer) 생성 → MSTR 리소스를 타겟으로 등록
VPC Endpoint Service 생성 → NLB를 연결
Allow Principals에 Databricks 계정 추가
2 Databricks 측 Account Console → Security → Networking → NCC
Private endpoint rules → Type: VPC Resource
MSTR의 VPC Endpoint Service name 입력
3 MSTR VPC 측 Databricks가 자동 생성한 VPC Endpoint 연결 승인 (Accept)

3. 정리

방향용도필요 여부구성 요약

(Frontend PrivateLink)		 MSTR이 SQL Warehouse에

결론: MSTR이 Databricks에 쿼리를 보내고 결과를 받아오는 일반적인 BI 용도라면, 방향 A(Frontend PrivateLink)만 구성하면 됩니다. Serverless SQL Warehouse도 동일한 workspace 엔드포인트를 사용하므로 구성이 동일합니다.

첫 번째 메일에서 MSTR 측이 안내한 과정(LGIT이 Endpoint Service 생성 → MCE가 Endpoint 생성)은 방향 B의 패턴인데, MSTR→Databricks 연결에서는 방향이 반대이므로 해당 과정은 불필요합니다.

네트워크 구조에 따라 세부 설정이 달라질 수 있으니, 필요하시면 아키텍처 리뷰 미팅을 잡아주시면 더 정확한 가이드를 드리겠습니다.

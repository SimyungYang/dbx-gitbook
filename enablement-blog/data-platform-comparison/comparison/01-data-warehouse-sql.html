데이터 웨어하우스 / SQL 분석

Databricks SQL + Photon이 전통 DW를 넘어서는 차세대 SQL 분석 경험을 제공합니다

핵심: Lakehouse가 전통 DW를 대체하는 이유

전통적 데이터 웨어하우스는 SQL 분석에 최적화되어 있지만, 데이터 레이크의 비정형 데이터나 ML 워크로드를 처리하려면 데이터를 복제해야 합니다. Databricks Lakehouse는 하나의 데이터 사본 위에서 SQL 분석, ML, 스트리밍을 동시에 실행하며, Photon 엔진으로 전통 DW 수준의 SQL 성능을 제공합니다.

아키텍처 비교

각 플랫폼의 SQL 분석 아키텍처 접근법

Databricks — Lakehouse SQL

  • 엔진: Photon (C++ 네이티브 벡터화 엔진)
  • 스토리지: Delta Lake on 고객 클라우드 스토리지
  • 아키텍처: 컴퓨팅-스토리지 완전 분리
  • 포맷: 오픈 포맷 (Delta/Iceberg UniForm)
  • 차별점: SQL + ML + 스트리밍이 동일 데이터 위에서 동작
  • 차별점: Genie Code로 자연어 SQL 분석 가능

Snowflake — Cloud DW

  • 엔진: 독점 MPP 엔진
  • 스토리지: Snowflake 관리형 (독점 포맷)
  • 아키텍처: 컴퓨팅-스토리지 분리 (내부)
  • 포맷: 독점 → Iceberg 전환 중
  • 제약: ML은 Cortex로 제한적 지원

AWS — Redshift + Athena

  • 엔진: Redshift (MPP), Athena (Presto/Trino)
  • 스토리지: Redshift RA3 + S3
  • 아키텍처: 서비스 분리 (DW와 Lake 쿼리 별도)
  • 제약: Redshift와 Athena 간 경험 분절

GCP — BigQuery

  • 엔진: Dremel (서버리스)
  • 스토리지: BigQuery Storage (독점) + BigLake
  • 아키텍처: 완전 서버리스
  • 제약: ML은 Vertex AI로 분리됨

Databricks SQL이 선택받는 이유

Photon 엔진

C++ 네이티브 벡터화 실행 엔진으로 기존 Spark SQL 대비 최대 12배 성능. TPC-DS 100TB에서 업계 최고 수준의 가격 대비 성능을 달성합니다.

Liquid Clustering

파티셔닝의 진화. 데이터 레이아웃을 자동 최적화하여 수동 OPTIMIZE나 Z-ORDER 없이도 최적의 쿼리 성능을 유지합니다.

Predictive I/O

AI가 쿼리 패턴을 학습하여 데이터 프리페치, 캐싱, 스케줄링을 자동 최적화. 관리자 개입 없이 성능이 지속적으로 향상됩니다.

Genie Code + AI/BI

비즈니스 사용자가 자연어로 질문하면 SQL/Python 코드를 자동 생성하고 실행. 분석가의 SQL 지식 없이도 셀프서비스 분석이 가능합니다.

SQL + ML 통합

동일한 SQL Warehouse에서 AI_QUERY, AI_GENERATE 등의 AI 함수를 SQL 내에서 직접 호출. 별도 ML 파이프라인 없이 SQL 분석에 AI를 결합합니다.

비용 투명성

스토리지는 고객 클라우드(S3/ADLS/GCS) 직접 과금, 컴퓨팅은 DBU 기반 초 단위 과금. 유휴 시 자동 종료로 낭비 제로.

결론: 데이터 웨어하우스 / SQL 분석

SQL 분석만 필요하다면 모든 플랫폼이 경쟁력 있습니다. 하지만 SQL 위에서 AI를 결합하고, 자연어 분석으로 접근성을 높이며, 오픈 포맷으로 벤더 종속을 탈피하고 싶다면 —

Databricks Lakehouse SQL이 최적입니다

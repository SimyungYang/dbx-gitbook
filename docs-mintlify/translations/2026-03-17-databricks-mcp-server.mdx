- MCP를 활용하여 Agent Bricks와 AI Playground로 에이전트를 빌드하고 배포하세요
- 자체 MCP 서버를 호스팅하여 도구를 갖춘 에이전트를 빌드하고 배포하세요
- UC Functions, Genie 스페이스, Vector Search 인덱스를 위한 Databricks 관리형 서버를 활용하여 에이전트를 데이터에 연결하세요

MCP(Model Context Protocol)는 LLM에 도구를 제공하는 표준으로서 최근 몇 달간 업계 전반에 걸쳐 폭발적인 관심을 받고 있습니다. 이는 LLM이 행동을 취하는 데 필요한 맥락을 보다 자연스러운 형태로 제공하기 때문에 중요한 진보입니다. 그러나 기업들은 거버넌스, 보안, 탐색성을 희생하지 않으면서 MCP를 어떻게 활용할 수 있을지 고민하고 있습니다. 에이전트에 비즈니스 맥락을 제공하면서도 데이터를 안전하게 보호해야 하는 상황에서, 인증과 인가, 접근 권한, 제어에 대한 의문들이 쏟아지고 있습니다.

우리는 MCP를 수용하고 이를 Unity Catalog와 Mosaic AI의 강점과 결합하게 되어 기쁩니다. 이를 통해 세 가지의 장점을 모두 얻을 수 있습니다: 에이전트가 행동을 취할 수 있게 하는 **MCP**, 에이전트를 빌드하고 평가하기 위한 **Mosaic AI**, 그리고 거버넌스와 탐색을 위한 **Unity Catalog**. 이제 조직의 보안과 거버넌스를 존중하는 방식으로 에이전트에 데이터 인텔리전스를 제공할 수 있습니다.

이번 출시를 통해, 우리는 MCP의 어려운 부분을 대신 처리해 드립니다: 관리형 서버는 기본적으로 온-비하프-오브-유저(on-behalf-of-user) 인증을 지원하여, Unity Catalog에서 이미 구축한 거버넌스를 그대로 존중합니다. 또한 Databricks Apps에서 OAuth를 표준으로 지원하여 손쉬운 호스팅이 가능하며, Playground는 빠르게 프로토타이핑할 수 있는 안전한 환경입니다.

## Databricks 관리형 서버로 안전한 데이터 접근

첫 번째 관리형 서버 세트를 통해 Genie, Vector Search, UC Functions로 Databricks의 데이터에 안전하게 접근할 수 있습니다. 엔터프라이즈급 보안을 염두에 두고 설계된 관리형 MCP 서버는 사용자의 권한을 자동으로 준수합니다. 이를 통해 Unity Catalog에서 모든 데이터를 계속 거버닝하면서 MCP를 활용할 수 있으며, 관리와 거버넌스를 위한 별도의 장소가 필요하지 않습니다.

통신 회사에서 지원 담당자를 돕는 고객 지원 에이전트를 구축한다고 가정해 보겠습니다. Unity Catalog의 정형 및 비정형 데이터를 활용하여 에이전트를 더 스마트하게 만들 수 있습니다:

1. **Genie**를 통해 계정 및 요금제 정보와 같은 정형 데이터에 접근할 수 있습니다
2. **Vector Search**를 통해 지원 문서 및 지식 베이스 문서와 같은 비정형 데이터에 접근할 수 있습니다
3. **UC Functions**를 통해 청구 관련 질문에 답변하기 위한 결정적(deterministic) 함수를 구축할 수 있습니다

![Databricks Managed MCP Servers](https://www.databricks.com/sites/default/files/inline-images/image5_24.png?v=1750206659)

이러한 서버는 별도의 유지 관리가 필요 없이 완전히 관리됩니다. MCP 표준이 계속 발전함에 따라 최신 사양을 지원하도록 업데이트됩니다. 이를 통해 조직에 중요한 일에 에너지를 집중할 수 있습니다.

## Databricks Apps로 자체 MCP 서버를 안전하게 호스팅

통신 회사 지원 에이전트를 계속 구축해 보겠습니다: 현재 장애 상황을 확인하고 새로운 장애를 보고할 수 있는 내부 API가 있습니다. 그러나 이 모든 정보가 인프라에 갇혀 있어 에이전트가 접근할 수 있는 명확한 방법이 없습니다.

이때 **Databricks Apps**가 에이전트에 생명을 불어넣어 줍니다: 기본 제공되는 OAuth 지원, 간편한 Git 기반 배포, 내장된 권한 및 거버넌스를 통해, 레거시 서비스와 API를 단 몇 분 만에 MCP 서버로 전환할 수 있습니다. Databricks Apps는 서버리스 인프라 위에 직접 구축되어 있으므로, 에이전트 사용량이 늘어나도 확장성에 대해 걱정할 필요가 없습니다.

![MCP Telco Outage Server](https://www.databricks.com/sites/default/files/inline-images/image3_50.png?v=1750206659)

지금 바로 [Marketplace의 간편한 MCP 서버 템플릿](https://marketplace.databricks.com/details/e109cf23-ee53-46ce-92d7-8a562138c62c/Databricks_Custom-MCP-Server-Template)을 사용해 보세요.

## Agent Bricks 또는 AI Playground로 MCP 에이전트 빌드 및 배포

Databricks는 MCP를 활용한 에이전트 빌드와 배포를 쉽게 만들어 줍니다. Agent Bricks에서 Supervisor Agent는 Databricks Apps에서 구축한 MCP 서버를 지원하여, 버튼 클릭 한 번으로 에이전트를 내부 서비스에 연결할 수 있습니다.

![Configure agents](https://www.databricks.com/sites/default/files/inline-images/image6_14.png?v=1750206659)

AI Playground를 사용하여 MCP를 활용한 에이전트를 프로토타이핑할 수도 있으며, 코드를 작성하기 전에 MCP 서버와 에이전트 로직을 모두 테스트할 수 있습니다. 도구가 활성화된 LLM에서 "Tools" 드롭다운을 사용하면 관리형 서버나 Databricks Apps를 통한 커스텀 서버를 간편하게 사용해 볼 수 있습니다.

![AI playground](https://www.databricks.com/sites/default/files/inline-images/image1_16.gif?v=1750206659)

Databricks를 사용하여 에이전트를 배포하면, MLflow와 Agent Evaluation을 통해 어떤 도구가 어떻게 사용되고 있는지 쉽게 확인할 수 있습니다. 이를 통해 사용 사례의 요구에 맞게 에이전트 로직이나 도구 로직을 조정할 수 있으며, 에이전트의 동작에 대한 완전한 가시성을 확보할 수 있습니다:

![Traces](https://www.databricks.com/sites/default/files/inline-images/image4_33.png?v=1750206659)

## 향후 확장 계획

이번 출시에 대해 매우 기대하고 있으며, 이것은 시작에 불과합니다. 향후 DBSQL과 같은 다른 유형의 Databricks 리소스에 대한 관리형 서버 지원을 확대할 예정입니다. 또한 다른 기업과 서비스가 제공하는 MCP 서버를 관리, 탐색, 거버닝하기 위한 카탈로그 지원을 구축하여, 현재 사용하고 있는 곳에서 바로 중앙 집중화할 수 있도록 할 계획입니다.

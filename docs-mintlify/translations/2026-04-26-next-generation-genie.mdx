**요약**
- **개별 Genie Space에서 통합 어시스턴트로**: 새 Genie는 단일 Genie Space의 범위를 넘어, 검증된 모든 Genie Space, 거버넌스 대시보드, Databricks Apps의 비즈니스 로직을 활용해 도메인 간 질문에 답변합니다.
- **Unity Catalog AI Gateway로 비정형 데이터까지 확장**: Google Drive, SharePoint와 같은 외부 비정형 소스에 대한 빌트인 커넥터가 도입되어, 거버넌스를 유지하면서 정형/비정형 데이터를 함께 질의할 수 있습니다.
- **Databricks One — 통합 비즈니스 사용자 홈**: 워크스페이스 경계를 없앤 단일 홈 화면, 도메인 기반 자산 그룹화, Entra·Okta 자동 ID 관리, 100,000명 이상 사용자 지원, 모바일 네이티브 앱(iOS/Android)을 제공합니다.

대부분의 조직에서 데이터는 흩어져 있고, 그 결과 인사이트도 흩어집니다. 부서마다 자신만의 데이터 질문이 있고, 자신만의 대시보드가 있고, 결국 자신만의 "진실의 버전"이 생깁니다. 비즈니스 사용자가 정말 묻고 싶은 것은 사일로를 가로지르는 질문—"우리 가장 가치 있는 고객들이 이번 분기 어떻게 행동하고 있고, 그게 공급망 결정에 어떻게 영향을 줘야 하는가?"—이지만, 그런 질문에 답하려면 관계자 다섯 명, 도구 세 개, 그리고 며칠의 시간이 필요했습니다.

다음 세대 Databricks Genie는 이 문제를 정면으로 해결합니다. 개별 Genie Space에서 일하는 도구가 아니라, **조직의 모든 검증된 비즈니스 로직 위에서 작동하는 단일 통합 어시스턴트**가 되었습니다.

## 새로운 Genie의 핵심 변화

### 사전 모델링 없이도 정확한 답변

기존 Genie는 데이터셋마다 별도의 Genie Space를 구성하고, 각 Space에 검증된 메트릭과 예시 질문을 등록해야 했습니다. 새 Genie는 **이미 조직에 존재하는 비즈니스 로직을 재사용**합니다:

- 검증된 Genie Spaces의 의미론적 정의
- 거버넌스 대시보드의 차원·메트릭
- Databricks Apps에 인코딩된 비즈니스 규칙

이 모든 것을 최신 추론 모델과 에이전틱 아키텍처가 종합하여, 도메인을 가로지르는 복잡한 질문에 답합니다. 한 고객은 "통합 채팅의 에이전틱 추론은 전체 생태계에 흩어진 먼 관계들을 합성한다"고 표현했습니다.

### 엔터프라이즈 커넥터 — 정형 데이터를 넘어

비즈니스 결정의 상당 부분은 정형 데이터만으로는 답할 수 없습니다. 계약서, 회의록, 정책 문서, 시장 보고서 같은 비정형 자료에 핵심 맥락이 있습니다.

빌트인 커넥터를 통해 Genie는 이제 **Google Drive, SharePoint** 같은 외부 비정형 소스에 직접 접근합니다. 이 모든 접근은 Unity Catalog AI Gateway를 거쳐 **거버넌스되고 감사됩니다**. 누가 어떤 문서에 어떻게 접근했는지가 일관된 정책으로 통제됩니다.

### Databricks One — 통합 비즈니스 사용자 홈

이전에 "Databricks One"으로 미리 공개되었던 통합 홈이 새 Genie의 진입점이 됩니다. 비즈니스 사용자는 더 이상 워크스페이스가 어떻게 나뉘어 있는지 알 필요가 없습니다:

- **자동화된 ID 관리**: Microsoft Entra ID, Okta와의 SCIM 동기화로 100,000명 이상의 사용자를 손쉽게 온보딩
- **도메인 기반 자산 그룹화**: 워크스페이스가 아니라 비즈니스 개념(예: "마케팅", "공급망")으로 자산을 그룹화
- **커스텀 회사 URL**: 조직 고유의 도메인으로 접근 가능
- **단일 로그인**: 한 번 로그인하면 회사 전체의 인사이트에 접근
- **일관된 거버넌스**: Unity Catalog가 모든 접근에 동일한 정책을 적용

### 모바일 액세스

매장 직원, 현장 영업, 물류 매니저처럼 데스크톱 앞에 앉아 있을 시간이 없는 사용자들을 위해, **iOS·Android 네이티브 앱**이 출시되었습니다. 대시보드, Apps, 채팅 인터페이스 전부를 모바일에서 사용할 수 있습니다.

## 왜 중요한가

새 Genie의 본질은 "더 똑똑한 챗봇"이 아니라, **조직 전체의 검증된 데이터 자산을 단일한 대화형 인터페이스로 통합**한다는 데 있습니다. 비즈니스 사용자는 사일로의 존재를 알 필요가 없고, 데이터 팀은 같은 메트릭을 여러 번 정의할 필요가 없습니다. 거버넌스는 조직이 기존에 만들어둔 정책 위에서 자동으로 적용됩니다.

## 시작하기

새 Genie UI, 통합 채팅 경험, 계정 수준 기능에 대한 문서와 리소스가 공개되었습니다. 자세한 내용은 [Databricks 문서](https://docs.databricks.com/aws/en/genie/)를 참고하세요.

## 🧭 이 기능은 무엇이고 언제 쓰나

Databricks의 3-tier Admin 모델을 이해합니다. **Account Admin** > **Metastore Admin** > **Workspace Admin** 순으로 권한 범위가 다릅니다.

## ⚙️ 핵심 옵션 / 파라미터

| 옵션 | 설명 |
|------|------|
| 본 Lab에서 다루는 핵심 변수 | 샘플 코드의 `<...>` 부분을 본인 환경에 맞게 교체 |

## 💻 샘플 코드

```text
Account Admin   : Account-level (모든 Workspace, Metastore, Cloud Resource)
                  - Workspace 생성/삭제
                  - User/SP/Group 생성
                  - Audit Log Delivery
                  - Compliance Profile

Metastore Admin : UC Metastore (한 Region 내 모든 Catalog)
                  - Catalog/Schema/Table 등 모든 UC 객체 SUPER USER
                  - Storage Credential / External Location 생성
                  - System Tables 권한 부여

Workspace Admin : 단일 Workspace 내 모든 권한
                  - Cluster Policy / SQL Warehouse 관리
                  - 모든 사용자 노트북 접근
                  - 그러나 UC 객체 권한은 별도
```

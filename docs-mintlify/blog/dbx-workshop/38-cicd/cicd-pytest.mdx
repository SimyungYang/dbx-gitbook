본 Lab은 **PyTest 통합** 을 단일 기능 단위로 직접 수행하며 동작을 체감하는 것이 목적입니다. `M38. CI/CD — DAB + GitHub Actions` 의 핵심 흐름 안에서 이 단계가 어떤 역할을 하는지 위치를 잡습니다.

|------|------|
| 본 Lab에서 다루는 핵심 변수 | 샘플 코드의 `<...>` 부분을 본인 환경에 맞게 교체 |

## 💻 샘플 코드

```yaml
# .github/workflows/databricks-dab.yml
name: DAB Deploy
on:
  push: { branches: [main] }
  pull_request:
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: databricks/setup-cli@main
      - run: databricks bundle validate --target dev
        env:
          DATABRICKS_HOST: ${{ secrets.DATABRICKS_HOST }}
          DATABRICKS_TOKEN: ${{ secrets.DATABRICKS_TOKEN }}
```

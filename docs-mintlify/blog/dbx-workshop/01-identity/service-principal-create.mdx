## 🧭 이 기능은 무엇이고 언제 쓰나

Service Principal (SP)은 사람이 아닌 자동화/앱이 사용하는 ID입니다. CI/CD, Job 실행, SDK 자동화에 필수입니다.

## ⚙️ 핵심 옵션 / 파라미터

| 옵션 | 설명 |
|------|------|
| 본 Lab에서 다루는 핵심 변수 | 샘플 코드의 `<...>` 부분을 본인 환경에 맞게 교체 |

## 💻 샘플 코드

```bash
# CLI로 SP 생성
databricks account service-principals create --json '{
  "display_name": "workshop-sp",
  "active": true
}'

# 또는 Account Console UI에서 생성
```

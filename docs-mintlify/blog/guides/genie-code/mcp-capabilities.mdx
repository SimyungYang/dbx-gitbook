## MCP로 Genie Code가 할 수 있게 되는 일

### 1. Unity Catalog Functions → 커스텀 비즈니스 로직 실행

UC에 등록한 SQL/Python 함수를 Genie Code가 자연어 요청에 따라 자동 호출합니다.

> **참고**
UC Function의 `COMMENT` 가 Genie Code가 도구를 선택할 때 참고하는 설명이 됩니다.


### 2. Vector Search → RAG 기반 문서 검색

사내 문서, 기술 문서, 매뉴얼을 검색하여 답변에 활용합니다.

### 3. Genie Space → 자연어 데이터 분석

코딩 작업 중에 데이터를 직접 분석할 수 있습니다.

> **성공**
Vector Search + Genie Space + UC Function을 모두 연결하면, 복합 작업을 한 번의 요청으로 처리할 수 있습니다.


### 4. UC Connection (외부 MCP) → 외부 서비스 연동

| 외부 서비스 | 프롬프트 예시 |
|---------|-----------|
| **GitHub** | "우리 리포에서 ETL 관련 코드 찾아줘" |
| **Slack** | "이 분석 결과를 #analytics에 공유해줘" |
| **JIRA** | "이 버그에 대한 JIRA 티켓 만들어줘" |

### 5. Databricks App (커스텀 MCP) → 사내 시스템 연동

자체 개발한 MCP 서버로 사내 ERP, CRM, HR 시스템 등을 연결합니다.

---

## MCP 연결 전후 비교

| 항목 | MCP 연결 전 | MCP 연결 후 |
|------|----------|----------|
| **데이터 접근** | 현재 노트북/파일만 | 워크스페이스 전체 + 외부 시스템 |
| **외부 서비스** | 불가 | GitHub, Slack, JIRA 등 |
| **역할** | 코딩 어시스턴트 | **종합 업무 자동화 에이전트** |

---

## 실전 MCP 구성 예시: GitHub + Slack + Databricks SQL

```
프롬프트: "@orders 테이블에서 지난 주 대비 주문량이 30% 이상 감소한 지역을 찾고,
         GitHub에서 관련 ETL 코드의 최근 변경사항을 확인해서,
         Slack #data-ops에 분석 결과를 공유해줘."
```

## MCP 보안 고려사항

| 보안 영역 | Databricks의 대응 |
|----------|------------------|
| **인증 정보 보호** | Unity Catalog Connection이 자격 증명을 안전하게 관리 |
| **최소 권한 원칙** | OAuth 스코프 제한 가능 |
| **데이터 유출 방지** | Managed MCP Proxy가 트래픽 중계 및 감사 로그 기록 |
| **사용자별 인증** | Per-user OAuth로 개별 인증 지원 |
| **도구 실행 감사** | Unity Catalog 감사 로그에 MCP 호출 기록 |

## 활용 예시: GitHub MCP 서버

**설정 순서:**

1. GitHub App 생성 → Callback URL: `https://<workspace-url>/login/oauth/http.html`
2. Unity Catalog Connection 생성 → Host: `https://api.githubcopilot.com`, Base path: `/mcp`
3. Genie Code에서 연결 추가

**사용 가능한 도구**: `search_code`, `get_file_contents`

## 기타 외부 서비스

| 서비스 | 활용 시나리오 |
|--------|-------------|
| **Slack** | 작업 완료 후 채널에 결과 알림 |
| **JIRA** | 분석 중 발견한 이슈를 티켓으로 생성 |
| **Notion** | 분석 결과를 위키에 자동 기록 |
| **Genie Space** | 자연어로 데이터 분석 (MCP를 통해 호출) |

> **팁**
인기 MCP 서버 전체 목록과 실전 시나리오는 [인기 MCP 서버 & 활용 시나리오](../mcp/popular-servers)에서 확인하세요.


### 제한 사항

| 제한 | 상세 |
|------|------|
| **Agent 모드 전용** | MCP 서버는 Agent 모드에서만 사용 가능 |
| **도구 수 제한** | 전체 MCP 서버에 걸쳐 **최대 20개 도구** |
| **전송 방식** | 외부 MCP 서버는 Streamable HTTP만 지원 |

## MCP 비용 구조

| 리소스 | 비용 유형 |
|--------|----------|
| Unity Catalog Functions | Serverless General Compute |
| Genie Spaces | Serverless SQL Compute |
| Vector Search Indexes | Vector Search 가격 |
| Custom MCP Servers | Databricks Apps 가격 |

> **참고**
MCP 프로토콜 자체에 대한 추가 비용은 없습니다. 도구가 많을수록 LLM 프롬프트에 포함되는 토큰이 증가하므로, 실제로 사용하는 도구만 활성화하세요.

## MCP 에이전트 개발 베스트 프랙티스

1. **도구 이름 하드코딩 금지** — 런타임에 동적 탐색
2. **출력 파싱 금지** — 결과 해석은 LLM에 위임
3. **LLM 기반 도구 선택** — 사용자 요청에 따라 자동 결정

---

## MCP 활용 성숙도 모델

| 단계 | 수준 | 사용 패턴 |
|------|------|----------|
| **Level 1** | 기본 | Agent 모드만 사용 (MCP 없이) |
| **Level 2** | 연결 | Managed MCP 1-2개 추가 |
| **Level 3** | 확장 | External MCP 2-3개 추가 (Slack, GitHub, JIRA) |
| **Level 4** | 맞춤 | Custom MCP 개발 (사내 시스템 래핑) |

> **팁**
대부분의 팀은 Level 1에서 시작하여 2-3개월에 걸쳐 Level 3까지 도달합니다.

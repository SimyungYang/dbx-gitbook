## 리전 개요

Databricks AWS는 전세계 여러 리전에서 운영됩니다. 대부분의 기능은 모든 리전에서 사용 가능하지만, 일부 기능은 특정 리전에서만 지원됩니다.

<Note>
**Cross-Geography Routing이란?** 서울 리전에서 특정 AI 기능을 사용할 때, 요청이 US 리전의 모델 서빙 인프라로 라우팅됩니다. 데이터는 처리 후 반환되며 외부에 저장되지 않지만, 데이터 거주(Data Residency) 요구사항이 있는 경우 주의가 필요합니다.
</Note>

### Cross-Geography Routing 필요 (서울 → US 처리)

<Warning>
아래 기능은 서울 리전에서 사용 가능하지만, 요청이 US 리전으로 라우팅됩니다. 2026년 2월부터 일본/한국 리전 모델 호스팅이 GA되어 일부 기능은 cross-geo 없이 사용 가능합니다.
</Warning>

| 기능 | 상태 | 비고 |
|------|------|------|
| Foundation Model APIs (Pay-per-token) | ✅ GA | US로 라우팅 |
| ai_query (배치 추론) | ✅ GA | US로 라우팅 |
| ai_parse_document v2 | ✅ GA | US로 라우팅 |
| ai_prep_search | ✅ GA | US로 라우팅 |
| ai_extract / ai_classify v2 | Public Preview | US로 라우팅 |
| 기타 AI Functions | ✅ GA | US로 라우팅 |

### Genie 서울 리전 In-Region 모델 호스팅 (2026.02 GA)

2026년 1월 30일부터 Genie 전용 LLM이 서울 리전에 배포되었으며, **2월에 Regional Model Hosting이 GA**되었습니다.

<Warning>
**Cross-Geo 설정과 데이터 거주 트레이드오프**

서울 리전에서 Genie를 **in-region으로만** 처리하려면 워크스페이스의 **Cross-Geography Processing을 OFF**해야 합니다. 하지만 이 설정은 워크스페이스 전체에 적용되므로 다른 Designated Services에 영향을 줍니다:

| Cross-Geo OFF 시 | 결과 |
|-------------------|------|
| Genie | ✅ 서울 리전 내 처리 (in-region) |
| Foundation Model APIs (PT) | ✅ 한국/일본 리전으로 라우팅 |
| AI Functions (ai_query 등) | ✅ 한국/일본 리전으로 라우팅 |
| Pay-per-token (PPT) | ✅ 한국/일본 리전으로 라우팅 |
| Databricks Assistant / Genie Code | ❌ cross-geo 필요 기능은 비활성화 |
| Knowledge Assistant / Supervisor Agent | ❌ 비활성화 |

**권장 방안**: 데이터 거주 규제가 엄격한 고객(금융권, 통신사 등)은 Genie 전용 워크스페이스를 분리하여 cross-geo OFF로 운영하고, 일반 업무용 워크스페이스는 cross-geo ON을 유지하는 것이 좋습니다. Delta Sharing으로 Genie 워크스페이스에 데이터를 피딩하는 구성도 가능합니다.

참고: [Designated Services 문서](https://docs.databricks.com/aws/en/resources/designated-services)
</Warning>

### 서울 Partner-Powered AI In-Country (2026.03 GA)

2026년 3월부터 아래 AI 기능이 **서울 리전에서 데이터 라우팅 보장** (cross-geo ON 상태에서도)과 함께 사용 가능합니다:

- Genie Code
- AI-generated comments
- AI/BI 대시보드 AI 기능

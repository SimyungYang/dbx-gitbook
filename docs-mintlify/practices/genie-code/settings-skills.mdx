> 이 문서는 **[Genie Code](/practices/genie-code/overview)** 섹션의 일부입니다.


Skills는 **재사용 가능한 작업 템플릿** 입니다. "이 작업을 할 때는 이 절차를 따라라"라는 레시피입니다. Agent 모드에서만 동작합니다.

**판단 기준**: 같은 유형의 작업을 반복하면서 매번 동일한 절차를 프롬프트에 작성하고 있다면, 스킬로 만드세요.

---

## 폴더 구조

**Workspace 스킬** (전체 공유):
```
/Workspace/.assistant/skills/{skill-name}/
├── SKILL.md           # 필수
├── template.py        # 선택
└── examples/          # 선택
```

**User 스킬** (개인용):
```
/Workspace/Users/{email}/.assistant/skills/{skill-name}/SKILL.md
```

## SKILL.md 작성법

```markdown
---
name: data-quality-check
description: 테이블의 데이터 품질을 검증하고 리포트를 생성합니다
---

# 데이터 품질 검증 스킬

## 이 스킬을 사용하는 경우
- 테이블 품질 확인, 새 소스 수집 후 검증, 정기 리포트

## 1단계: 프로파일링
행 수, 컬럼 수, null 비율, 고유값 수, min/max/mean/std

## 2단계: 품질 규칙 점검
PK 중복, NOT NULL 위반, 날짜 범위, 숫자 범위

## 3단계: 리포트
요약(통과율 %) + 상세(규칙별 통과/실패) + 권장 조치
```

## 실전 스킬 예시

### 파이프라인 템플릿

```markdown
---
name: pipeline-template
description: Bronze/Silver/Gold DLT 파이프라인 코드 생성
---
필수 입력: 소스, 대상 카탈로그, 증분 여부, 스케줄
규칙: Bronze(원본 수집, _ingested_at 추가) → Silver(정제, _processed_at) → Gold(집계)
모든 테이블 @dlt.table(), Expectations 포함, Workspace 네이밍 컨벤션 준수
```

### 대시보드 생성

```markdown
---
name: dashboard-builder
description: AI/BI 대시보드 구성
---
1단계: 스키마 확인, 날짜/차원/측정값 식별
2단계: 상단 KPI → 중단 트렌드 → 하단 테이블
3단계: 위젯별 독립 SQL, 날짜 필터, 주석 포함
4단계: AI/BI 대시보드 발행, 팀 VIEW 권한
```

### 품질 모니터링

```markdown
---
name: quality-monitor
description: Unity Catalog 모니터 설정 + 알림 규칙
---
1단계: 스키마/데이터 특성, timestamp 컬럼, 핵심 컬럼 선정
2단계: Lakehouse Monitor 생성 (매일 08시)
3단계: 커스텀 메트릭 (null 비율, 범위 이탈, 고유값 변화율)
4단계: 알림 (WARNING >5% / CRITICAL >20%)
```

## Workspace vs User 스킬

| 기준 | Workspace | User |
|------|-----------|------|
| 사용 범위 | 팀 전체 동일 절차 | 개인 특화 반복 작업 |
| 표준화 | 필수 (파이프라인, 배포) | 불필요 (개인 분석) |
| 관리 | 관리자 버전 관리 | 개인 자유 수정 |

> **팁**
User 스킬로 검증 후 팀에 유용하면 Workspace로 승격하세요.

## 워크샵 실습 환경 통합

[**핸즈온(SmartTV) 워크샵**](/workshop/overview)의 실습 환경이 **Genie Code + AI Dev Kit MCP** 단일 환경으로 통합되었습니다. 기존의 Track A/B/C 분기가 제거되고, 모든 실습이 동일한 환경에서 진행됩니다.

- **단일 실습 환경** — Genie Code와 AI Dev Kit MCP가 연결된 상태를 전제로 모든 모듈이 재작성되었습니다. 별도 트랙을 선택할 필요 없이 바로 실습을 시작할 수 있습니다
- **Custom Instructions에 MCP 규칙 추가** — `manage_genie`, `manage_dashboard` 등 MCP 도구 활용 규칙이 기본 Custom Instructions에 포함되어, MCP 도구 호출 실패 시 자동 재시도하도록 안내합니다
- **범용 교육 자료화** — 특정 기업 참조가 모두 제거되어 누구나 활용할 수 있는 범용 교육 콘텐츠로 정리되었습니다

## AI Dev Kit MCP 도구 가이드 보강

[**Genie Code + AI Dev Kit 구성 가이드**](/workshop/section-3-ai-tools-setup/genie-code-aidevkit)에 실무 활용에 필요한 상세 가이드가 추가되었습니다.

- **작업별 MCP 도구 매핑** — 데이터 조회, 파이프라인 생성, 대시보드 구성 등 10가지 작업에 필요한 필수/보조 도구를 표로 정리했습니다
- **Warm-up 패턴** — Cold start 대응을 위해 "현재 사용자 정보 알려줘"로 MCP 서버를 미리 활성화하는 팁 추가
- **Fallback 대응** — MCP 도구 호출 실패 시 "다시 MCP 도구로 시도해줘" 프롬프트로 복구하는 방법, `execute_code`와 `execute_sql`의 타임아웃 차이 안내

## 개선 사항

- **핸즈온 모듈 메타데이터 통일** — [Section 4](/workshop/section-4-smart-tv-handson/01-setup) 전체 모듈 상단에 소요시간·사전조건·실습환경·MCP 도구가 표 형식으로 일관되게 표시됩니다
- **AI Builder App 가이드 개선** — [Section 2](/workshop/section-2-ai-builder-app/overview) 가이드가 초심자 친화적으로 재작성되어, 완료 후 AI Dev Kit 연결 페이지로 바로 이동할 수 있습니다
- **프롬프트 원칙 수정** — "작업마다 새 대화 시작" 원칙이 "맥락 전환 시 새 대화 시작"으로 [수정](/workshop/section-1-vibe-coding-intro/genie-code/prompt-principles)되었습니다. 같은 테이블의 연속 작업은 같은 대화에서 이어가는 것이 효율적입니다

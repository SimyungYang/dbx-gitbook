## 가이드: RAG 구현 파이프라인 (6편)

[**RAG 가이드**](/blog/guides/rag/overview)에 검색 증강 생성 시스템의 전체 구현 과정을 다루는 6편의 실전 가이드가 공개되었습니다. 이전에 공개된 [핵심 개념](/blog/guides/rag/concepts/overview)과 [고급 Retrieval 전략](/blog/guides/rag/advanced-retrieval/overview)에 이어, 실제 파이프라인을 처음부터 끝까지 구축하는 데 필요한 단계를 안내합니다.

- [**데이터 준비**](/blog/guides/rag/data-preparation) — PDF 파싱, 중복 제거, 메타데이터 관리 등 RAG 품질의 70%를 결정하는 전처리 과정을 다룹니다. `ai_parse_document`를 활용한 비정형 문서 처리와 Delta Table 기반 데이터 파이프라인 구성법을 안내합니다
- [**청킹 전략 상세**](/blog/guides/rag/chunking-strategies) — 고정 크기, 재귀적, 시맨틱 청킹 등 다양한 분할 전략의 트레이드오프를 비교하고, 문서 유형별 최적 청크 크기를 선택하는 기준을 제공합니다
- [**Vector Search 설정**](/blog/guides/rag/vector-search) — Databricks Vector Search의 HNSW 알고리즘 기반 서버리스 벡터 인덱스를 구성하는 방법을 안내합니다. Delta Sync Index, Direct Vector Access Index 등 인덱스 유형별 특성과 임베딩 모델 선택 가이드를 포함합니다
- [**RAG 체인 구축**](/blog/guides/rag/chain-building) — LangChain과 Databricks를 통합하여 Retriever → Augmenter → Generator로 이어지는 RAG 체인을 구현하는 방법을 다룹니다. Foundation Model API 연동과 프롬프트 설계 패턴을 포함합니다
- [**RAG 평가**](/blog/guides/rag/evaluation) — MLflow Evaluate와 RAGAS 프레임워크를 활용하여 검색 품질과 생성 품질을 분리 평가하는 방법을 안내합니다. Faithfulness, Context Relevance 등 핵심 메트릭의 의미와 활용법을 설명합니다
- [**RAG 배포**](/blog/guides/rag/deployment) — 평가를 통과한 RAG 체인을 Databricks Model Serving으로 프로덕션에 배포하고, 서버리스 스케일링과 MLflow Tracing 기반 모니터링을 구성하는 방법을 다룹니다

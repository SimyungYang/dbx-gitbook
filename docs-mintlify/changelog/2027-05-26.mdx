## 업데이트

- **Apps SP 권한 관리 — 실전 패턴 3종 추가** — [Apps 개요](/blog/guides/apps/overview) 가이드에 서비스 프린시펄(SP) 권한을 간소하게 관리하는 패턴이 정리되었습니다. 앱마다 `GRANT`를 반복 실행하는 대신, 상황에 맞는 방식을 고를 수 있습니다.
  - **패턴 1 · UC 그룹 (추천)** — 앱 전용 그룹에 권한을 한 번만 부여하고, 새 앱의 SP를 그룹에 추가하면 모든 권한이 자동 상속됩니다. 앱이 여러 개일수록 관리 부담이 크게 줄어듭니다. `readonly` / `readwrite` / `admin` 등 용도별로 그룹을 나누면 최소 권한 원칙도 지킬 수 있습니다.
  - **패턴 2 · 자동화 스크립트** — 앱 생성 후 한 번 실행하면 엔타이틀먼트, SQL Warehouse `CAN_USE`, UC 카탈로그 권한을 일괄 설정하는 `grant-app-permissions.sh` 예제가 제공됩니다. CI/CD 파이프라인에 바로 연결할 수 있습니다.
  - **패턴 3 · app.yaml 리소스 선언** — `app.yaml`에 Warehouse·Serving Endpoint를 선언하면 해당 리소스 권한이 SP에 자동 부여됩니다. 앱 1~2개 규모에서 빠르게 적용하기 좋습니다.
- **"흔한 실수" 알림 강화** — [앱 만들기](/blog/guides/apps/create-app) 가이드의 권한 부여 경고 박스에 UC 그룹 패턴 크로스 레퍼런스가 추가되어, 여러 앱을 운영할 때 어떤 방식을 선택해야 하는지 한눈에 확인할 수 있습니다

## 가이드: 인증(Authentication) 심화 시리즈

[**인증 종합 가이드**](/blog/guides/platform-setup/authentication/overview)의 3개 심화 페이지를 소개합니다. PAT에서 OAuth로, 그리고 Token Federation까지 — Databricks 인증의 전체 스펙트럼을 실무 관점에서 다룹니다.

- [**OAuth U2M (사용자 인증)**](/blog/guides/platform-setup/authentication/oauth-u2m) — PKCE 흐름 기반의 대화형 인증. 브라우저 로그인 한 번으로 CLI·SDK·IDE에서 1시간 수명 토큰이 자동 발급·갱신됩니다. PAT를 디스크에 저장하지 않아도 되는 구조적 보안 개선
- [**OAuth M2M (서비스 인증)**](/blog/guides/platform-setup/authentication/oauth-m2m) — Service Principal 기반의 완전 자동화 인증. CI/CD 파이프라인, 스케줄 Job, IaC 등 사람 개입 없이 동작하는 워크로드를 위한 표준 방식. Secret 회전, Vault 연동, 멀티 언어 코드 예제 포함
- [**PAT & Token Federation**](/blog/guides/platform-setup/authentication/pat-federation) — PAT의 올바른 사용 범위와 한계, 그리고 OAuth Token Federation을 통해 GitHub Actions·Azure DevOps·AWS STS에서 Secret 없이 Databricks에 인증하는 방법

## 가이드: Serverless NCC — 서버리스 환경의 프라이빗 연결

[**Serverless NCC**](/blog/platform-setup/aws/serverless-ncc) 가이드가 추가되었습니다. Serverless SQL Warehouse, Serverless Notebook, Model Serving 등 Databricks 관리 VPC에서 실행되는 워크로드가 고객의 S3 버킷이나 VPC 리소스에 프라이빗하게 접근하는 방법을 안내합니다.

- **NCC vs Classic PrivateLink 비교** — 적용 대상, VPC 관리 주체, 구성 난이도를 한눈에 비교. 두 방식은 상호 보완적이며, 보안 요구사항에 따라 함께 구성할 수 있습니다
- **S3 Private Endpoint & VPC Resource Endpoint** — S3 버킷 단위의 프라이빗 엔드포인트와 NLB 경유 VPC 리소스 접근 설정을 단계별로 안내
- **Stable IP 지원** — NCC에 고정 IP를 활성화하여 방화벽 허용 목록에 활용하는 방법

## 가이드: Databricks Geo — 데이터 거주 관리

[**Databricks Geo**](/blog/guides/platform-setup/databricks-geos) 가이드에서 데이터 주권 규제에 대응하는 Databricks의 Geo 제도를 다룹니다. GDPR, 한국 개인정보보호법, 중국 PIPL 등 각국 규제 요구사항에 맞춰 데이터 처리 위치를 제어하는 방법을 안내합니다.

- **Geo별 리전 매핑** — AWS·Azure·GCP 리전이 어느 Geo에 속하는지 확인하고, 워크스페이스의 데이터 처리 범위를 파악할 수 있습니다
- **Cross-Geo 제어** — 기본적으로 데이터는 워크스페이스와 동일한 Geo 내에서만 처리되며, Cross-Geo 처리는 명시적 활성화가 필요합니다
- **AI 서비스 데이터 경로** — AI 어시스턴트, Foundation Model API 등 관리형 서비스의 데이터 처리 위치가 Geo 경계 내에서 보장되는 방식을 설명합니다

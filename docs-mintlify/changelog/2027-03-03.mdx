## 새 기능

- **Genie Space 데이터 구성 가이드** — [테이블 및 지식 저장소 구성](/blog/guides/genie-space/data-config) 페이지가 [Genie Space 운영 가이드](/blog/guides/genie-space/overview)에 추가되었습니다. Knowledge Store의 구성 요소(테이블 설명, 동의어, SQL Expression, 조인 관계, 프롬프트 매칭)를 우선순위별로 설정하는 방법과, 컬럼 메타데이터가 Genie의 SQL 생성 정확도에 미치는 영향을 상세히 설명합니다
- **Genie Space 테스트 & 벤치마크** — [테스트 & 벤치마크](/blog/guides/genie-space/testing) 페이지가 추가되었습니다. 벤치마크 기능을 활용하여 인스트럭션이나 테이블 변경 후 기존 질문의 정확도가 유지되는지 회귀 테스트하는 방법을 안내합니다. 샘플 질문 설계 기준과 Correct/Incorrect 판정 기반의 반복 개선 사이클도 포함됩니다
- **Agent Bricks 베스트 프랙티스** — [베스트 프랙티스](/practices/agent-bricks/best-practices) 페이지가 [Agent Bricks 가이드](/practices/agent-bricks/overview)에 추가되었습니다. Knowledge Assistant의 문서 준비·청크 사이즈·인스트럭션 작성 기준, Genie Spaces의 테이블 설계·인스트럭션 최적화, Supervisor Agent의 라우팅 설계와 프로덕션 체크리스트를 컴포넌트별로 정리합니다
- **Databricks Apps app.yaml 설정** — [app.yaml 설정](/practices/apps/app-yaml) 페이지가 [Databricks Apps 가이드](/practices/apps/overview)에 추가되었습니다. `command`, `env`, `resources` 3개 섹션의 구조와 역할, 리소스 유형별(SQL Warehouse, Serving Endpoint, Secret) 설정 예시, 환경별(dev/staging/prod) 설정 분리 전략을 다룹니다

## 교육자료

- **MLOps 핸즈온 시작하기 & 팀 협업** — [시작하기](/hands-on/predictive-maintenance/getting-started) 페이지에서 소스코드 다운로드와 워크스페이스 가져오기 방법을 안내하고, [팀 협업 가이드](/hands-on/predictive-maintenance/00-team-collaboration) 모듈에서 여러 명이 동시에 ML 모델을 개발할 때 발생하는 클러스터 충돌, 자원 경쟁, 데이터 오염 문제의 해결 방법을 다룹니다

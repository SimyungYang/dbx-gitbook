## GenAI 도구 생태계 종합 가이드

[**GenAI 도구 생태계**](/blog/guides/genai-tools/overview) 가이드가 6개 카테고리 심화 페이지로 공개되었습니다. 각 도구의 동작 원리, 한계, Databricks 연동 포인트를 다룹니다.

- [**AI 코딩 어시스턴트**](/blog/guides/genai-tools/coding-assistants) — GitHub Copilot, Cursor, Claude Code, Windsurf의 3단계 파이프라인(컨텍스트 수집 → LLM 추론 → 코드 적용) 비교 및 Agent 모드 진화 방향
- [**LLM API & 플랫폼**](/blog/guides/genai-tools/llm-platforms) — GPT-4.1, Claude 4, Gemini 2.5, Llama 4 등 주요 모델의 특징과 Foundation Model API를 통한 Databricks 연동
- [**노코드 AI 빌더**](/blog/guides/genai-tools/ai-builders) — Dify, n8n, Flowise, LangFlow 등 비개발자도 AI 워크플로를 구축할 수 있는 도구 안내
- [**AI 생산성 & 검색**](/blog/guides/genai-tools/ai-productivity) — Perplexity, NotebookLM, Notion AI 등 일상 업무 자동화 도구
- [**오픈소스 LLM**](/blog/guides/genai-tools/opensource-llm) — Ollama, vLLM, Hugging Face를 활용한 자체 호스팅 및 데이터 주권 확보 전략
- [**AI 관측성 & 평가**](/blog/guides/genai-tools/ai-observability) — MLflow Tracing, LangSmith, W&B, Phoenix를 활용한 LLM 앱 품질 보장 및 디버깅 방법

## 교육자료: SQL AI 함수 & Metric Views

**교육자료** 탭의 데이터 웨어하우징·AI/BI 섹션에서 두 가지 실전 가이드를 소개합니다.

### AI 함수 — SQL에서 직접 LLM 호출

[**AI 함수**](/training/06-data-warehousing/ai-functions) 가이드에서 SQL 쿼리 한 줄로 텍스트 분류, 감정 분석, 정보 추출을 수행하는 방법을 다룹니다.

- `ai_query()` — 자유 형식 프롬프트로 LLM을 호출하는 범용 함수
- `ai_classify()`, `ai_extract()`, `ai_summarize()` 등 특화 함수 사용법
- Foundation Model API 또는 외부 모델(OpenAI, Anthropic) 엔드포인트 연동

### Metric Views — 비즈니스 시맨틱 레이어

[**Metric Views**](/training/08-ai-bi/metric-views) 가이드에서 "같은 지표인데 팀마다 숫자가 다른" 문제를 해결하는 방법을 안내합니다.

- 매출, 활성 사용자, 이탈률 등 비즈니스 지표를 **Unity Catalog에서 중앙 정의**
- 대시보드, SQL, Genie, 외부 BI 도구에서 동일한 지표를 일관되게 사용
- 거버넌스(권한, 리니지, 감사)가 자동 적용되는 Databricks 네이티브 시맨틱 레이어

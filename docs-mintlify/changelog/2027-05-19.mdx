## 업데이트

- **Apps 서비스 프린시펄 권한 관리 실전 패턴 추가** — [Apps 개요](/blog/guides/apps/overview)에 앱마다 반복되는 `GRANT` 작업을 줄이는 3가지 패턴(UC 그룹 활용, 자동화 스크립트, `app.yaml` 리소스 선언)이 추가되었습니다. 앱 전용 UC 그룹을 한 번만 만들어 두면 이후 새 앱의 SP를 그룹에 추가하는 것만으로 모든 권한이 상속되어, 앱이 많아질수록 권한 관리 부담이 크게 줄어듭니다. 용도별 그룹 분리(readonly/readwrite/admin) 팁과 앱 생성 후 한 번 실행하는 권한 부여 스크립트 예제도 함께 제공합니다. [앱 생성 가이드](/blog/guides/apps/create-app)에도 UC 그룹 패턴 크로스 레퍼런스가 연결되었습니다

## 버그 수정

- **Apps 아키텍처 설명 정정 — Control Plane에서 Serverless Compute Plane으로** — [Apps 개요](/blog/guides/apps/overview)와 [네트워킹](/blog/guides/apps/networking) 문서에서 Databricks Apps가 Control Plane에서 실행된다고 기술된 부분이 정정되었습니다. 실제로는 Model Serving, Vector Search와 동일한 **Serverless Compute Plane**에서 컨테이너로 실행되며, Control Plane은 OAuth 인증과 URL 라우팅만 담당합니다. 인증 완료 후 모든 트래픽은 Serverless Compute Plane으로 직접 전달됩니다. 앱의 실행 환경과 네트워크 특성을 정확히 이해하고 설계에 반영할 수 있습니다

## 핸즈온: 가전 IoT (공기청정기 + 에어컨) E2E 핸즈온

[**Genie Code Vibe 워크샵**](/workshop/overview)에 가전 IoT 핸즈온이 추가되었습니다. 공기청정기(PuriCare)와 에어컨(DUALCOOL/WHISEN)의 IoT 센서 데이터를 기반으로 Databricks 핵심 기능을 체험하는 6개 모듈입니다.

- [**환경 설정**](/workshop/section-5-appliance-handson/01-setup) — Unity Catalog 네임스페이스 구성, Genie Code Agent Mode 준비, Custom Instructions 설정
- [**공기청정기 데이터 생성**](/workshop/section-5-appliance-handson/02-air-purifier-data) — PM2.5·VOC·CO2 센서, 필터 상태, ThinQ 원격 제어 등 8개 테이블 100만 건 생성
- [**에어컨 데이터 생성**](/workshop/section-5-appliance-handson/03-air-conditioner-data) — 온도·습도 센서, 컴프레서 이벤트, 에너지 효율, 정비 이력 등 9개 테이블 120만 건 생성
- [**SDP 파이프라인**](/workshop/section-5-appliance-handson/04-sdp-pipeline) — 17개 Bronze 테이블의 Medallion Architecture 변환, Expectations 기반 데이터 품질 검증
- [**대시보드 & Genie Space**](/workshop/section-5-appliance-handson/05-dashboard-genie) — 공기청정기·에어컨·통합 모니터링 대시보드 3종, 도메인 특화 Genie Space 3종
- [**에이전트 개발**](/workshop/section-5-appliance-handson/06-agent-development) — Knowledge Assistant(제품 FAQ) + Genie Agent(IoT 분석) + Supervisor Agent 멀티 에이전트 구성

기존 Smart TV 케이스와 함께, 소속 조직에 맞는 핸즈온을 선택하여 진행할 수 있습니다.

## 가이드: Agent Bricks — 선언형 AI 에이전트 빌더

[**Agent Bricks 실전 가이드**](/blog/guides/agent-bricks/overview)가 5개 페이지로 구성되었습니다. 코드 없이 프로덕션 수준의 AI 에이전트를 구축·평가·배포하는 전 과정을 다룹니다.

- [**Knowledge Assistant**](/blog/guides/agent-bricks/knowledge-assistant) — Unity Catalog 데이터 소스를 연결하여 사내 문서 기반 Q&A 에이전트를 선언형으로 구성
- [**Genie Spaces 연동**](/blog/guides/agent-bricks/genie-agent) — Genie Space를 에이전트의 도구로 통합하여 자연어 데이터 분석을 에이전트 워크플로우에 포함
- [**Supervisor Agent**](/blog/guides/agent-bricks/supervisor) — 여러 하위 에이전트를 오케스트레이션하는 멀티 에이전트 아키텍처 구성
- [**평가 & 베스트 프랙티스**](/blog/guides/agent-bricks/evaluation) — MLflow Evaluate 기반 품질 측정, Synthetic Task Generation을 통한 자동 테스트, 운영 노하우

## 가이드: RAG 핵심 개념 & 고급 Retrieval 전략

[**RAG 핵심 개념**](/blog/guides/rag/concepts/overview) 시리즈 6개 페이지에서 검색 증강 생성의 기반 기술을 수식 수준까지 설명합니다.

- [**임베딩**](/blog/guides/rag/concepts/embedding) · [**유사도 측정**](/blog/guides/rag/concepts/similarity) — 텍스트를 벡터로 변환하고, 코사인 유사도·내적·L2 거리로 비교하는 원리
- [**벡터 인덱스 & ANN**](/blog/guides/rag/concepts/vector-index) — 수백만 벡터에서 밀리초 안에 유사 벡터를 찾는 HNSW, IVF 등의 자료 구조
- [**BM25**](/blog/guides/rag/concepts/bm25) · [**하이브리드 검색**](/blog/guides/rag/concepts/hybrid-search) · [**Reranking**](/blog/guides/rag/concepts/reranking) — 키워드 검색과 Dense 검색을 결합하는 RRF 알고리즘, Cross-Encoder 기반 정밀 재순위

[**고급 Retrieval 전략**](/blog/guides/rag/advanced-retrieval/overview) 7개 페이지에서는 프로덕션 RAG의 검색 품질을 높이는 실전 기법을 다룹니다.

- [**하이브리드 검색**](/blog/guides/rag/advanced-retrieval/hybrid-search) — Dense + Sparse 결합, 다국어 토크나이저 고려사항
- [**전처리 & 후처리**](/blog/guides/rag/advanced-retrieval/preprocessing) — Query Rewrite, HyDE, 컨텍스트 압축
- [**고급 청킹**](/blog/guides/rag/advanced-retrieval/chunking-advanced) — Semantic Chunking, Parent-Child 구조, 슬라이딩 윈도우
- [**한국어 RAG 최적화**](/blog/guides/rag/korean-nlp) — 교착어 특성, 형태소 분석 기반 토큰화, 한국어 특화 임베딩 모델 선택

[**Lakebase 교육자료**](/training/11-lakebase/what-is-lakebase) 8개 페이지에서 Databricks의 PostgreSQL 호환 관리형 OLTP 데이터베이스를 학습할 수 있습니다.

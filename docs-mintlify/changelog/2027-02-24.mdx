## 새 기능

- **Genie Code 사용법 상세 가이드** — [Genie Code 사용법](/blog/guides/genie-code/usage) 페이지가 공개되었습니다. Chat 모드와 Agent 모드의 차이와 전환 기준, 인라인 코드 제안 및 Quick Fix 활용법, Diagnose Error를 통한 에러 진단 자동화, `/fix`·`/explain` 등 Slash 명령어 사용법을 한 곳에서 확인할 수 있습니다
- **Genie Space 모니터링 & 피드백** — [모니터링 및 피드백](/blog/guides/genie-space/monitoring) 페이지가 [Genie Space 운영 가이드](/blog/guides/genie-space/overview)에 추가되었습니다. 모니터링 탭으로 질문 추이·응답 정확도·Thumbs Up/Down 지표를 추적하고, 사용자 피드백을 Instructions·데이터 모델 업데이트로 연결하는 개선 사이클을 안내합니다. 배포 후 2주간 집중 모니터링으로 사용자 이탈을 방지하는 운영 전략도 포함됩니다
- **Databricks Apps 리소스 & 환경변수** — [리소스 & 환경변수](/blog/guides/apps/resources) 페이지가 [Databricks Apps 가이드](/blog/guides/apps/overview)에 추가되었습니다. SQL Warehouse, Model Serving Endpoint, Secret 등 리소스 유형별 `app.yaml` 설정법과 런타임 자격 증명 자동 주입 원리를 다루며, 하드코딩 없이 환경(dev/staging/prod)별 설정을 안전하게 분리하는 방법을 설명합니다

## 교육자료

- **분산 학습 상세** — [분산 학습](/training/09-machine-learning/distributed-training) 교육자료가 추가되었습니다. Data Parallelism, Model Parallelism, Pipeline Parallelism 3가지 전략의 트레이드오프를 비교하고, Databricks ML Runtime에서 TorchDistributor를 활용한 PyTorch 분산 학습 실습을 안내합니다
- **RAG 파이프라인** — [RAG 파이프라인](/training/10-agent-development/rag-pipeline) 교육자료가 추가되었습니다. 오프라인 파이프라인(문서 수집 → 파싱 → 청킹 → Delta 저장 → Vector Search 인덱싱)과 온라인 파이프라인(질문 → 검색 → Reranker 재순위화 → LLM 생성 → 응답)의 전체 구조를 실습 중심으로 설명합니다
- **Lakebase Data Sync** — [Data Sync](/training/11-lakebase/data-sync) 교육자료가 추가되었습니다. Lakebase(OLTP)와 Delta Lake(OLAP) 간 WAL 기반 CDC 동기화로 별도 ETL 없이 변경 사항을 자동 반영하는 아키텍처와, 실시간 대시보드·ML 피처 갱신에 즉시 활용하는 시나리오를 다룹니다

## 업데이트

- **탭 순서 재편 — [교육자료](/intro/overview) 우선 배치** — 상단 탭 순서가 체계적인 학습 콘텐츠를 먼저 보여주도록 [교육자료](/intro/overview) → [실전 가이드](/blog/overview) 순으로 바뀌었습니다. Databricks를 처음 접하는 사용자는 커리큘럼에서 시작하고, 특정 기능 구현이 필요한 사용자는 실전 가이드로 바로 이동할 수 있습니다
- **핸즈온 2개 탭 → [워크샵](/hands-on/workshop/overview) 단일 탭으로 통합** — 기존 "Genie Code 핸즈온"과 "MLOps 핸즈온"이 [워크샵](/hands-on/workshop/overview) 탭 하나로 합쳐졌습니다. 풀데이·하프데이 타임테이블과 모든 실습 자료를 한곳에서 확인할 수 있습니다
- **56개 긴 페이지 → 156개 서브페이지로 분할** — 읽기 시간 5분 이내를 목표로 Apps, Agent Bricks, Lakebase, Unity Catalog, Data Engineering 등 주요 가이드의 긴 페이지가 기초·상세·심화 단위로 재구성되었습니다. 필요한 부분만 빠르게 찾아 읽을 수 있습니다
- **Apps 가이드 재편** — [SP(서비스 프린시펄) 권한 관리](/practices/apps/service-principals)가 Apps 개요에서 분리되어 독립 페이지로 정리되었고, 프레임워크별 예제도 [Streamlit](/practices/apps/examples/streamlit)·[Gradio](/practices/apps/examples/gradio)·[FastAPI](/practices/apps/examples/fastapi)·[AppKit](/practices/apps/examples/appkit)·[Agent UI](/practices/apps/examples/agent-ui)·[Lakebase](/practices/apps/examples/lakebase)·[트러블슈팅](/practices/apps/examples/troubleshooting) 7개 파일로 나뉘었습니다. 새로 추가된 [앱 업데이트 & 버전 관리](/practices/apps/updating) 가이드에서는 `databricks apps deploy` 재배포, 의존성·`app.yaml` 변경 시 동작 등 운영 단계에서 자주 묻는 시나리오를 다룹니다
- **플랫폼 비교 독립 그룹화 & 토픽 재배치** — [플랫폼 비교](/blog/overview)가 AI/BI에서 독립 그룹으로 분리되었고, Metric Views는 Unity Catalog → AI/BI로, Lakebase는 AI 에이전트 뒤 → 데이터 웨어하우징 뒤로 이동했습니다. 전 탭이 **기초 → 상세 → 심화** 순서로 재정렬되어 학습 난이도에 따라 따라가기 쉬워졌습니다
- **교육자료 ↔ 실전 가이드 크로스 링크 강화** — 교육자료 13개 섹션에 overview 페이지가 연결되었고, 개념을 학습한 뒤 동일 주제의 실전 구현 가이드로 바로 이동할 수 있도록 양방향 크로스 링크가 추가되었습니다

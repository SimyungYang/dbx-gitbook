## 새로운 기능

- **AI Dev Kit Windows 정식 지원** — [Genie Code + AI Dev Kit 구성 가이드](/hands-on/workshop/section-3-ai-tools-setup/genie-code-aidevkit)에 Windows 설치 경로(winget / Chocolatey / Scoop)와 Git Bash·WSL 환경에서의 실행 방법이 추가되었습니다. 기존 macOS·Linux 중심 가이드에서 벗어나 Windows 사용자도 별도 우회 없이 MCP 앱을 배포·연결할 수 있습니다
- **`deploy.sh`에 `--profile` · `--app-name` 옵션 추가** — Databricks CLI 프로필을 명시적으로 지정하거나 앱 이름을 커스터마이즈할 수 있게 되었습니다. 여러 워크스페이스를 오가는 구성원이나 동일 워크스페이스에 여러 인스턴스를 운영하는 팀이 한 번의 명령으로 원하는 대상에 배포할 수 있습니다
- **`uninstall.sh` 제거 스크립트 신설** — 배포된 앱, 업로드된 소스, 등록된 스킬을 한 번에 정리하는 공식 제거 스크립트가 추가되었습니다. 워크샵 종료 후나 재설치가 필요한 상황에서 수동 정리 없이 환경을 깨끗하게 되돌릴 수 있습니다

## 업데이트

- **`--catalog` 옵션 필수화** — `deploy.sh`에서 하드코딩되어 있던 기본 카탈로그가 제거되고 `--catalog`가 필수 인자가 되었습니다. 실수로 다른 워크스페이스의 카탈로그에 배포되던 문제가 사라지고, 배포 대상이 명령어에 명시적으로 드러납니다
- **Python 3.10+ 필수** — AI Dev Kit MCP 서버의 최소 Python 요구 버전이 3.9에서 3.10으로 상향 조정되었습니다. `deploy.sh`가 `python3.13` → `python3.10` 순으로 자동 탐색하여 설치된 최신 인터프리터를 사용합니다
- **README 영어·한국어 이중 제공** — AI Dev Kit 저장소 README가 영어(기본)와 한국어(`docs/README_ko.md`) 두 버전으로 분리되었습니다. 상단 링크로 두 언어를 바로 전환할 수 있어, 해외 구성원과 자료를 공유하거나 이중 언어 환경에서 온보딩을 진행할 때 별도 번역 없이 활용할 수 있습니다
- **README 탭 렌더링 GitHub 호환** — GitHub에서 보이지 않던 GitBook 전용 `{% tabs %}` 문법이 `<details>` 태그로 교체되어, 저장소 페이지에서 설치·배포 단계가 접힘 블록으로 정상 표시됩니다

## 버그 수정

- **Windows Git Bash(MINGW64) 경로 변환 문제 해결** — Git Bash 환경에서 `/Workspace/...` 경로가 `C:/Program Files/Git/Workspace/...`로 자동 변환되어 `databricks workspace import-dir`가 실패하던 문제가 `deploy.sh`와 `uninstall.sh`에서 해결되었습니다. Windows에서 별도 환경 변수 설정 없이 바로 동작하며, macOS·Linux 사용자에게는 영향이 없습니다
- **Windows Git Bash 인증 자동 복구** — `databricks current-user me`가 프로필 없이 실패할 때(Git Bash/MINGW64 환경에서 빈번) DEFAULT 프로필로 자동 재시도하도록 보완되었습니다. 실패 시 안내 메시지에 프로필 힌트가 함께 노출됩니다
- **앱 배포 경로 오류 수정** — `apps deploy --source-code-path`에 로컬 경로가 전달되어 배포가 실패하던 문제가 수정되었습니다. 이제 소스를 먼저 워크스페이스에 업로드한 뒤 워크스페이스 경로를 참조하여 정상 배포됩니다

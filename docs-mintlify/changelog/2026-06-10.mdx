## 교육자료: AI Guardrails & Review App

**교육자료** 탭의 AI 에이전트 섹션에 프로덕션 배포에 필수적인 2가지 가이드가 추가되었습니다.

### AI Guardrails — 에이전트 입출력 보호

[**AI Guardrails**](/training/10-agent-development/guardrails) 가이드에서 AI 에이전트의 입력·출력을 자동으로 필터링하는 방법을 다룹니다.

- **AI Gateway 프록시** — 모든 LLM 호출에 안전 필터를 일괄 적용하는 중앙 게이트웨이 설정
- **Databricks Safety Filter** — 유해 콘텐츠 감지 및 차단 구성
- **PII 감지 & 마스킹** — 개인정보 자동 탐지 및 익명화 처리
- **토픽 차단 & 키워드 필터** — 허용/차단 목록 기반 대화 범위 제어
- **커스텀 Guardrails** — Llama Guard 등을 활용한 맞춤형 보호 규칙 구축

### Review App — 에이전트 베타 테스트

[**Review App**](/training/10-agent-development/review-app) 가이드에서 에이전트를 프로덕션에 배포하기 전 팀 내 테스트와 피드백 수집 체계를 안내합니다.

- `agents.deploy()`로 자동 생성되는 웹 기반 채팅 UI를 통한 베타 테스트
- 테스터 관리, 피드백 수집(긍정/부정, 텍스트, 기대 응답), 평가 데이터셋 변환
- Inference Table 스키마를 활용한 지속적 개선 루프 구축

## Genie Code 활용 레퍼런스

[**Genie Code 가이드**](/blog/guides/genie-code/overview)에 실무 활용도를 높이는 2가지 레퍼런스 페이지를 소개합니다.

### 기능 레퍼런스

[**사용법**](/blog/guides/genie-code/usage) 페이지에서 Genie Code의 모든 기능을 한곳에서 확인할 수 있습니다.

- **패널 모드** — Side·Docked·Floating 3가지 레이아웃과 키보드 단축키
- **Chat vs Agent 모드** — 동작 방식, 컨텍스트 범위, 사용자 개입 수준 비교
- **인라인 제안 & 코드 액션** — Ghost Text, Explain·Fix·Optimize·Test 선택 액션
- **Slash 명령어** — `/explain`, `/fix`, `/optimize`, `/test`, `/doc`, `/findTables`, `/findRelevantCode` 등 8가지 명령어
- **`@` 멘션** — 테이블·컬럼 자동완성으로 정확한 데이터 참조

### 실전 시나리오 & ROI

[**활용 시나리오**](/blog/guides/genie-code/scenarios) 페이지에서 작업별 예상 시간 절감 효과와 구체적인 프롬프트 예시를 확인할 수 있습니다.

- **Top 5 ROI 패턴** — Agent Mode EDA(2시간→5분), 대시보드 자동 생성(4시간→15분) 등
- **8가지 상세 시나리오** — 데이터 사이언스 노트북, SQL 최적화, AI/BI 대시보드, Lakeflow 파이프라인, Job 디버깅, MLflow GenAI 디버깅, 멀티 프로덕트 워크플로우

## Lakeflow 증분 처리 심화 가이드

[**Lakeflow 증분 처리**](/blog/guides/lakeflow-incremental/overview) 가이드의 3개 심화 페이지를 소개합니다. Full Refresh 방지 전략부터 CDC 운영까지 실무에 필요한 모든 내용을 다룹니다.

- [**Streaming Table**](/blog/guides/lakeflow-incremental/streaming-table) — 체크포인트 기반 증분 처리 원리, Full Refresh를 유발하는 5가지 조건, Blue/Green 안전 리프레시 패턴
- [**Materialized View**](/blog/guides/lakeflow-incremental/materialized-view) — Enzyme 엔진의 증분 vs 전체 재계산 판단 로직, 지원 연산자 매트릭스, 최적화 전략
- [**CDC 체크리스트**](/blog/guides/lakeflow-incremental/cdc-checklist) — APPLY CHANGES를 활용한 SCD Type 1/2 구현, APPLY CHANGES FROM SNAPSHOT, 프로덕션 운영 체크리스트

## 핸즈온: 예지보전 MLOps 파이프라인

[**예지보전 MLOps 완전 가이드**](/blog/hands-on/predictive-maintenance/overview)가 추가되었습니다. Databricks Lakehouse 위에서 ML 모델의 개발부터 배포·모니터링까지 전체 MLOps 파이프라인을 11단계로 실습합니다.

- [**Feature Engineering**](/blog/hands-on/predictive-maintenance/02-feature-engineering) — Spark와 Pandas UDF를 활용한 피처 생성, Feature Store 등록까지 단계별 안내
- [**모델 학습 & 챌린저 검증**](/blog/hands-on/predictive-maintenance/03-model-training) — XGBoost 학습, SHAP 해석, Champion/Challenger 모델 비교 워크플로우
- [**이상탐지 & 모니터링**](/blog/hands-on/predictive-maintenance/07-anomaly-detection) — 비정형 이미지 기반 이상탐지(PatchCore)와 Lakehouse Monitoring을 통한 Data Drift 자동 감지
- [**MLOps Agent & Job 스케줄링**](/blog/hands-on/predictive-maintenance/09-mlops-agent) — AI Agent가 모니터링 결과를 해석하고 재학습을 트리거하는 자동화 파이프라인 구성

## 가이드: Prompt Engineering 시리즈

[**Prompt Engineering**](/blog/guides/genai-concepts/prompt-engineering/overview) 가이드 5개 페이지에서 LLM 프롬프트 설계의 기초부터 보안까지 체계적으로 다룹니다.

- [**기본 기법**](/blog/guides/genai-concepts/prompt-engineering/basics) — Zero-shot, Few-shot, Chain-of-Thought를 동일 예시로 비교하여 각 기법의 효과 차이를 직관적으로 확인
- [**고급 기법**](/blog/guides/genai-concepts/prompt-engineering/advanced) — ReAct, Tree of Thought, Self-Consistency 등 복잡한 추론 작업을 위한 전략
- [**한국어 프롬프트 최적화**](/blog/guides/genai-concepts/prompt-engineering/korean) — 존댓말/반말 지정, 토큰 효율, 코드 스위칭 등 한국어 특유의 프롬프트 작성 전략
- [**Prompt Injection 방어**](/blog/guides/genai-concepts/prompt-engineering/security) — 프롬프트 주입 공격의 원리와 System Prompt 수준의 방어 기법

## 가이드: A2A 프로토콜 — 에이전트 간 통신 표준

[**A2A (Agent-to-Agent Protocol)**](/blog/guides/genai-concepts/a2a-protocol/overview) 가이드에서 Google이 발표한 에이전트 간 개방형 통신 프로토콜을 다룹니다. MCP가 "도구 연결"이라면, A2A는 "에이전트 간 협업"을 위한 표준입니다.

- [**핵심 개념**](/blog/guides/genai-concepts/a2a-protocol/concepts) — Agent Card 구조, Task 라이프사이클(submitted → working → completed), MCP와의 역할 구분
- [**구현 가이드**](/blog/guides/genai-concepts/a2a-protocol/implementation) — 여행 예약 시나리오를 통한 A2A 기반 멀티 에이전트 협업 설계
- [**프로덕션 운영**](/blog/guides/genai-concepts/a2a-protocol/production) — 인증, 신뢰 경계, 보안 고려사항

## 가이드: Analytics Platform 비교

[**Analytics Platform 비교 가이드**](/blog/guides/platform-comparison/overview)에서 Databricks를 Snowflake, Redshift, BigQuery, Microsoft Fabric과 8개 핵심 영역에서 비교합니다. 플랫폼 도입이나 마이그레이션을 검토하는 의사결정자와 아키텍트에게 유용합니다.

- [**아키텍처**](/blog/guides/platform-comparison/architecture) · [**컴퓨트**](/blog/guides/platform-comparison/compute) · [**데이터 엔지니어링**](/blog/guides/platform-comparison/data-engineering) — 각 플랫폼의 설계 철학, 컴퓨트 분리 구조, ETL/ELT 접근 방식 비교
- [**SQL 분석**](/blog/guides/platform-comparison/sql-analytics) · [**ML/AI**](/blog/guides/platform-comparison/ml-ai) · [**거버넌스**](/blog/guides/platform-comparison/governance) — BI 워크로드, 머신러닝, 데이터 거버넌스 역량 비교
- [**가격 체계**](/blog/guides/platform-comparison/pricing) — DBU, 크레딧, 슬롯 등 각 플랫폼의 과금 모델과 비용 최적화 전략

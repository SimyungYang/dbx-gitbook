## 새 기능

- **AI 규제 & 컴플라이언스 가이드** — [AI 성숙도](/blog/guides/genai-concepts/ai-proficiency/overview) 섹션에 [AI 규제 & 컴플라이언스](/blog/guides/genai-concepts/ai-proficiency/regulation) 가이드가 추가되었습니다. EU AI Act(리스크 기반 4단계 분류, 최대 매출 7% 과징금)와 한국 AI 기본법의 핵심 요건을 정리하고, Databricks 환경에서 규제를 준수하는 방법(Unity Catalog 감사 로그, AI Gateway 가드레일, PII 마스킹)을 안내합니다. 금융·의료·법률·제조 등 업종별 규제 시나리오도 포함되어 있습니다
- **Claude Code 워크샵 가이드** — [AI 도구 환경 구성](/workshop/section-3-ai-tools-setup/overview) 섹션에 [Claude Code 사용하기](/workshop/section-3-ai-tools-setup/claude-code) 가이드가 추가되었습니다. Claude Code 설치부터 AI Gateway 경유 연결, MCP 구성까지 단계별로 안내하며, Genie Code와의 비교표(실행 환경, 데이터 접근, 비용 모델, 적합 작업)를 통해 상황에 맞는 도구를 선택할 수 있습니다
- **업종별 프롬프트 패턴 가이드** — [Prompt Engineering](/blog/guides/genai-concepts/prompt-engineering/overview) 섹션에 [실전 활용](/blog/guides/genai-concepts/prompt-engineering/practical) 가이드가 추가되었습니다. 금융(수치 정확성), 의료(면책 조항), 법률(조항 인용), 제조(안전 파라미터), 유통(실시간 데이터), 교육(힌트 제공) 등 6개 업종의 프롬프트 템플릿과 A/B 테스트 프레임워크를 제공합니다

## 개선 사항

- **Genie Code 전체 기능 레퍼런스** — [Genie Code 사용법](/blog/guides/genie-code/usage) 페이지가 430줄 규모의 종합 레퍼런스로 보강되었습니다. 패널 열기 3가지 방법(Sparkle 아이콘, `Cmd+Shift+;`, 셀 컨텍스트 메뉴), Chat Mode와 Agent Mode의 차이, 인라인 코드 제안, Quick Fix, Diagnose Error, Slash 명령어, `@` 테이블 참조 등 모든 기능을 한 페이지에서 확인할 수 있습니다
- **MLOps 팀 협업 가이드** — [예지보전 MLOps 핸즈온](/blog/hands-on/predictive-maintenance/overview)에 [팀 협업 가이드](/blog/hands-on/predictive-maintenance/00-team-collaboration)가 추가되었습니다. 워크스페이스 폴더 구조(개인 vs 공유), 클러스터 공유 시 주의사항, 테이블·모델 이름 충돌 방지, MLflow 실험 분리 방법 등 팀 단위 ML 개발에서 흔히 발생하는 문제와 해결 방법을 다룹니다
- **AI Dev Kit 독립 설치 가이드** — [AI 도구 환경 구성](/workshop/section-3-ai-tools-setup/overview) 섹션에 [AI Dev Kit 설치 & 구성](/workshop/section-3-ai-tools-setup/ai-dev-kit-setup) 가이드가 추가되었습니다. Claude Code 로컬 환경과 Genie Code MCP 환경 각각에 대해 44개 MCP 도구와 25개 Skills를 설치하는 방법을 안내합니다

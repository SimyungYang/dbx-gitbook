## Community Edition vs Full Trial

Databricks는 두 가지 무료 옵션을 제공합니다.

| 비교 항목 | Community Edition | Full Trial (14일) |
|-----------|-------------------|-------------------|
| **기간** | 무기한 | 14일 |
| **비용** | 무료 | 무료 (14일 이후 유료) |
| **클러스터** | 소규모 단일 노드만 | 다양한 크기의 클러스터 |
| **Unity Catalog** | ❌ | ✅ |
| **SQL Warehouse** | ❌ | ✅ (Serverless 포함) |
| **협업 기능** | 제한적 | 전체 기능 |
| **적합한 대상** | 개인 학습, 실습 | 팀 단위 평가, POC |

> **현업 권장**: PoC를 진행한다면, 반드시 **Full Trial**을 사용하세요. Community Edition에서는 Unity Catalog, SQL Warehouse, Serverless 등 핵심 기능을 체험할 수 없습니다.

## Account Console vs Workspace Console

Databricks에는 **두 가지 관리 콘솔**이 있습니다. 이 구분을 이해하는 것이 매우 중요합니다.

![Databricks Account and Workspace](https://docs.databricks.com/aws/en/assets/images/object-hierarchy-966dee8fa239626134ed6b7278beab2c.png)

> 출처: [Databricks 공식 문서](https://docs.databricks.com/aws/en/getting-started/high-level-architecture)

### Account Console (계정 콘솔)

**URL**: `https://accounts.cloud.databricks.com`

**누가 사용하나?** 조직의 **플랫폼 관리자** (Account Admin)가 사용합니다.

| 기능 | 설명 |
|------|------|
| **Workspace 관리** | Workspace 생성, 삭제, 설정 변경 |
| **사용자/그룹 관리** | 조직 전체의 사용자, 그룹, Service Principal 등록 (SCIM) |
| **Unity Catalog Metastore** | Metastore 생성 및 Workspace 할당 |
| **네트워크 설정** | VPC, PrivateLink, 방화벽 구성 |
| **청구/비용** | 사용량 조회, 결제 관리 |
| **감사 로그** | 계정 수준의 보안 감사 |
| **SSO 설정** | SAML/OIDC 인증 연동 |

> **한 마디로**: Account Console은 **"Databricks 자체를 관리"**하는 곳입니다. 일반 사용자는 접근할 필요가 없습니다.

### Workspace Console (워크스페이스 콘솔)

**URL**: `https://<workspace-name>.cloud.databricks.com`

**누가 사용하나?** **모든 Databricks 사용자** — 데이터 엔지니어, 분석가, 데이터 사이언티스트가 매일 사용합니다.

| 기능 | 설명 |
|------|------|
| **Notebook** | 코드 작성 및 실행 (Python, SQL, Scala, R) |
| **SQL Editor** | SQL 쿼리 작성 및 실행 |
| **Compute** | 클러스터, SQL Warehouse 생성/관리 |
| **Lakeflow (Jobs/Pipelines)** | 파이프라인 구성, Job 스케줄링 |
| **Unity Catalog 탐색** | 카탈로그/스키마/테이블 탐색, 권한 부여 |
| **AI/BI Dashboard** | 대시보드 생성, Genie Space 구성 |
| **ML** | MLflow 실험, Model Serving, Agent 개발 |
| **Repos** | Git 연동, CI/CD |

> **한 마디로**: Workspace Console은 **"실제 데이터 작업을 하는"** 곳입니다.

### 두 콘솔의 관계

```
Account Console (관리자 전용)
  ├── Workspace A (개발팀) ← 데이터 엔지니어가 매일 접속
  ├── Workspace B (분석팀) ← 분석가가 매일 접속
  └── Workspace C (프로덕션) ← CI/CD로 자동 배포
  
  + Unity Catalog Metastore (모든 Workspace에서 공유)
  + 사용자/그룹 (계정 수준에서 관리, Workspace에 할당)
```

| 질문 | 답변 |
|------|------|
| **"새 사용자를 추가하려면?"** | Account Console에서 추가 → Workspace에 할당 |
| **"새 테이블을 만들려면?"** | Workspace Console에서 SQL/Python으로 생성 |
| **"SSO를 설정하려면?"** | Account Console에서 SAML/OIDC 구성 |
| **"비용을 확인하려면?"** | Account Console의 Billing 메뉴 |
| **"쿼리를 실행하려면?"** | Workspace Console의 SQL Editor |

## 1. 클러스터 생성 및 시작

1. 좌측 메뉴에서 **Compute** 클릭
2. **Create Compute** 클릭
3. 클러스터 이름 입력 (예: `my-first-cluster`)
4. 나머지는 기본값 유지 → **Create Compute** 클릭
5. 약 3~5분 대기 (상태가 "Running"으로 변경되면 완료)

## 2. 노트북 생성

1. 좌측 메뉴에서 **+ New** → **Notebook** 클릭
2. 이름: `hello-databricks`
3. 기본 언어: **Python**
4. 방금 만든 클러스터를 연결

## 3. 코드 실행

```python
# 셀 1: Spark 버전 확인
print(f"Spark 버전: {spark.version}")
print("Hello, Databricks!")
```

```python
# 셀 2: 샘플 데이터 생성 및 분석
from pyspark.sql.functions import col, sum, avg, count

orders = spark.createDataFrame([
    (1, "노트북", "전자제품", 1200000, "2025-03-01"),
    (2, "키보드", "전자제품", 89000, "2025-03-01"),
    (3, "운동화", "패션", 159000, "2025-03-02"),
    (4, "텀블러", "생활용품", 25000, "2025-03-02"),
    (5, "모니터", "전자제품", 450000, "2025-03-03"),
], ["order_id", "product", "category", "price", "order_date"])

summary = orders.groupBy("category").agg(
    count("*").alias("주문건수"),
    sum("price").alias("총매출"),
    avg("price").alias("평균단가")
).orderBy(col("총매출").desc())

display(summary)
```

## 4. SQL로 분석

```python
orders.createOrReplaceTempView("orders")
```

```sql
%sql
SELECT
    order_date AS 주문일,
    COUNT(*) AS 주문건수,
    FORMAT_NUMBER(SUM(price), 0) AS 총매출
FROM orders
GROUP BY order_date
ORDER BY order_date
```

결과 아래 **차트 아이콘**을 클릭하고 Bar Chart를 선택하면 시각화됩니다.

## 5. Delta Lake 타임 트래블 체험

```python
orders.write.mode("overwrite").saveAsTable("main.default.trial_orders")
```

```sql
%sql
-- 데이터 수정
UPDATE main.default.trial_orders SET price = 999000 WHERE product = '노트북';

-- 변경 이력 확인
DESCRIBE HISTORY main.default.trial_orders;

-- 수정 전 데이터 조회 (버전 0 = 최초 상태)
SELECT * FROM main.default.trial_orders VERSION AS OF 0 WHERE product = '노트북';
-- price가 원래의 1200000으로 나옵니다!
```

> 이것이 Delta Lake의 **타임 트래블** 기능입니다. 실수로 데이터를 잘못 수정해도 과거 버전으로 되돌릴 수 있습니다.

## 반드시 시도할 5가지

14일은 생각보다 짧습니다. Full Trial에서 아래 5가지를 반드시 체험하세요.

| # | 활동 | 핵심 |
|---|------|------|
| 1 | **Unity Catalog** | 카탈로그.스키마.테이블 생성, GRANT 권한 부여 |
| 2 | **SQL Warehouse** | Serverless SQL Warehouse로 쿼리 실행 |
| 3 | **Auto Loader** | 클라우드 스토리지 파일 자동 수집 |
| 4 | **SDP 파이프라인** | Bronze → Silver → Gold 변환 |
| 5 | **Genie** | 자연어로 데이터 질의 |

## 체험판 제한사항

### Community Edition

| 제한 | 상세 |
|------|------|
| 클러스터 | 단일 노드(15GB)만, 멀티 노드 불가 |
| Unity Catalog | ❌ |
| SQL Warehouse | ❌ |
| Serverless | ❌ |
| SDP | ❌ |

### Full Trial (14일)

| 제한 | 상세 |
|------|------|
| 기간 | 14일 후 만료 (영업팀에 연장 요청 가능) |
| 크레딧 | 약 $400~$500 DBU |
| 일부 기능 | Private Link, BYOK 등 엔터프라이즈 기능 제한 |

> **크레딧 절약**: Auto-termination을 10분으로 설정하세요. 사용하지 않는 클러스터는 반드시 종료합니다.

## PoC에서 프로덕션으로

| 항목 | 체험판 | 프로덕션 |
|------|--------|---------|
| 네트워크 | 퍼블릭 | VPC / Private Link |
| 인증 | 이메일/비밀번호 | SSO (SAML/OIDC) |
| 데이터 저장 | Databricks 관리형 | 고객 관리형 S3/ADLS |
| 환경 분리 | 단일 Workspace | dev / staging / prod |
| 배포 | 수동 | Databricks Bundles (CI/CD) |

## 흔한 문제와 해결

| 문제 | 해결 |
|------|------|
| 클러스터 시작 안 됨 | 다른 인스턴스 타입 또는 Serverless 사용 |
| Permission denied | Unity Catalog 권한 확인 (GRANT) |
| samples 테이블 안 보임 | Community Edition에서는 없음 → Full Trial 필요 |
| 크레딧 빨리 소진 | Auto-termination 10분 설정 |
